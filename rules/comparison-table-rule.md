# 比較表作成ルール

## 目的

記事内で紹介しているサービス、アプリ、店舗、商品、選択肢などを、読者が比較しやすいようにHTMLテーブルで整理してください。

比較表は、読者の意思決定を助けるために設置します。
アフィリエイト記事の場合は、読者が自然におすすめサービスや公式サイトへ進めるように、CV導線も意識してください。

## 情報取得元

比較表の情報は、以下の優先順位で取得してください。

1. `articles/<記事フォルダ>/rewritten.html`
2. `articles/<記事フォルダ>/original.html`
3. 記事内に設置済みの公式サイトリンク・アフィリエイトリンク周辺の本文
4. 必要な場合のみ、公式サイトへアクセスして補足確認

ただし、公式サイトへのアクセスがブロックされたり、情報が明確に取得できない場合は、推測で埋めずに「追加確認が必要」と記載してください。

## 手動貼り付け禁止

以下のような手動HTML貼り付け前提の処理は禁止です。

```python
html_content = """
<!-- ここに実際のHTML記事内容を貼り付けてください -->
"""
```

比較表作成スクリプトでは、必ずファイルからHTMLを読み込んでください。

例：

* `articles/sample-article/rewritten.html`
* `articles/josousex-point/rewritten.html`
* `articles/sefure-busu/rewritten.html`
* `articles/newhalf-sex/rewritten.html`

## 抽出対象

以下を候補として抽出してください。

* ランキング内のH3見出し
* おすすめサービス一覧内のH3見出し
* アプリ紹介内のH3見出し
* 店舗・サービス・商品紹介のH3見出し
* H3見出し直後または近くにある `<a href="">` のリンク
* ボタンリンク、公式サイトリンク、詳細ページリンク、アフィリエイトリンク

ただし、以下は比較表候補から除外してください。

* FAQ内のH3
* 注意点だけを説明するH3
* まとめ内のH3
* 公式サイトやサービス名ではない見出し
* 内部アンカーリンク
* 目次リンク

## 比較表の列

記事ジャンルに応じて、比較項目を変えてください。

### アプリ・出会い系・サービス紹介記事の場合

以下を基本列にしてください。

* サービス名
* 特徴
* 料金・費用感
* 向いている人
* 注意点
* 公式サイト・詳細

### 店舗・スポット紹介記事の場合

以下を基本列にしてください。

* 名称
* 特徴
* エリア
* 向いている人
* 注意点
* 公式サイト・詳細


### その他の記事の場合

以下を基本列にしてください。

* 比較項目
* 特徴
* メリット
* 注意点
* 向いている人
* 詳細

## 情報不足時の対応

* 料金、費用、スペック、条件などが明確に抽出できない場合は「追加確認が必要」と記載してください。
* 推測で金額・速度・条件・効果を作らないでください。
* 記事本文内に根拠がない情報は断定しないでください。
* 公式サイトから取得できない情報も「追加確認が必要」としてください。
* 情報が少ない場合でも、空欄にはせず「追加確認が必要」と明記してください。

## HTML形式

比較表は、WordPressのコードエディターに貼り付け可能なHTMLで作成してください。

基本形式は以下です。

```html
<div style="overflow-x: auto; width: 100%; -webkit-overflow-scrolling: touch;">
  <table border="1" cellpadding="10" cellspacing="0" style="width: 100%; table-layout: fixed;">
    <thead>
      <tr>
        <th style="width: 150px;">項目名</th>
        <th style="width: 150px;">特徴</th>
        <th style="width: 150px;">料金・費用感</th>
        <th style="width: 150px;">向いている人</th>
        <th style="width: 150px;">注意点</th>
        <th style="width: 150px;">公式サイト・詳細</th>
      </tr>
    </thead>
    <tbody>
      <tr>
        <td><strong>サービス名</strong></td>
        <td>特徴</td>
        <td>料金・費用感</td>
        <td>向いている人</td>
        <td>注意点</td>
        <td><a href="URL" target="_blank" rel="noopener noreferrer">公式サイト</a></td>
      </tr>
    </tbody>
  </table>
</div>
```

## リンクルール

* 比較表内のリンクには、必ず `target="_blank"` と `rel="noopener noreferrer"` を付けてください。
* 既存記事内のアフィリエイトリンクや公式リンクがある場合は、原則としてそのURLを維持してください。
* URLを推測で作成しないでください。
* URLが取得できない場合は、リンクを無理に作らず「追加確認が必要」としてください。
* 外部リンクの扱いは `rules/external-link-rule.md` にも従ってください。

## 挿入位置

比較表は、以下の優先順位で挿入してください。

1. 「この記事でわかること」リストの直後
2. おすすめ・ランキング系H2の直前
3. 最初の比較・選び方系H2の直前

ただし、既に比較表がある場合は、重複して新規作成せず、既存比較表を補強してください。

## change-logへの記録

比較表を作成・挿入した場合は、`change-log.md` に以下を記録してください。

* 比較表を挿入した位置
* 抽出した候補数
* 表に入れた項目数
* 情報不足で「追加確認が必要」とした項目
* 公式サイトまたは外部ページへアクセス確認した場合、その結果
* 比較表を作成しなかった場合、その理由

## 検証

比較表追加後、以下を確認してください。

* 比較表が1回だけ設置されている
* 既存比較表がある場合は重複していない
* 見出し構造を壊していない
* WordPressに貼り付け可能なHTMLになっている
* 表内リンクに `target="_blank"` が入っている
* 表内リンクに `rel="noopener noreferrer"` が入っている
* 情報不足時に「追加確認が必要」と明記されている
* 推測で料金・条件・スペックを作っていない
* `change-log.md` に作業内容を記録している

## 品質修正ルール（必須）

### 挿入位置の禁止事項

比較表は「この記事でわかること」のcapbox内に絶対に挿入しないでください。

禁止：

* `<div class="swell-block-capbox">`、`cap_box`、`capbox` の中に比較表を入れる
* 「この記事でわかること」の `<ul>` / `<ol>` の直後で、同じcapbox内に入れる

正しい挿入位置：

1. 「この記事でわかること」のcapboxが完全に閉じた直後
2. おすすめ・ランキング系H2の直前
3. 既存比較表がある場合は新規追加せず、その表を補強する、または十分なら何もしない

### 比較対象の厳格化

比較表に入れてよいのは、原則として以下だけです。

* おすすめアプリ、サービス名、商品名、店舗名
* 公式サイトリンク・詳細リンク・アフィリエイトリンクが近くにある紹介項目
* ランキング・おすすめ・サービス紹介系H2配下のH3

以下は必ず除外してください。

* 選び方、注意点、FAQ、まとめ配下のH3
* 方法、手順、チェックリスト系のH3
* 「〜を選ぶ」「〜を確認」「〜のポイント」など、サービス名ではないH3
* 公式リンクが近くになく、サービス名・商品名・店舗名とも判断できない抽象見出し
* 「遊び目的の会員が多いアプリを選ぶ」「会員数が豊富で出会いやすいこと」「即日で会える機能があるか確認」「年齢層が近いユーザーが多いか」「完全無料アプリは避けよう」「セキュリティ対策がしっかりしているか」などの選定基準見出し

### 重複禁止と低品質表の抑止

`rewritten.html` 内に既存の比較表、または `comparison-table-block` がある場合、新しい比較表を追加しないでください。
既存比較表が十分なら何もしないでください。補強する場合も1つの比較表内で完結させます。

以下に該当する場合は比較表を作成しないでください。

* 候補の半数以上のセルが「追加確認が必要」になる
* サービス名ではない抽象項目が混ざる
* 公式サイト・詳細リンクが候補の半数未満しか取得できない
* 比較表として読者の意思決定に役立たない

作成しなかった場合は、必ず `change-log.md` に理由を記録してください。
