# SEOリライト基本ルール

## 絶対ルール

- 元記事の検索意図を変えない
- 既存の重要なH2/H3を勝手に削除しない
- 本文リライト前に必ずリライト計画を作る
- Codex Cloudで記事URLが渡された場合は、リライト計画作成後に確認待ちで止まらず本文リライトまで続行する
- 記事全体の情報量を減らさない
- 競合上位記事の構成を丸写ししない
- 読者にとって不要な説明は増やさない
- アフィリエイトリンクへの導線は自然にする
- 公共性や信頼性のある外部リンクを必要に応じて追加する
- 内部リンクを必要に応じて追加する
- h2/h3/h4の階層を崩さない
- WordPressに貼り付け可能なHTMLで出力する

## リライトの進め方

1. 元記事を分析する
2. リライト計画を作る
3. Codex Cloudで記事URLが渡された場合は確認待ちせず本文をリライトする
4. 変更内容を記録する
5. WordPress反映前のチェックを行う

## 水増し・汎用文の禁止

- H3タイトルだけを差し替えた汎用段落を量産しない
- 「〇〇については、〜」という定型文で各H3を埋めない
- 文字数を増やす目的で、番号だけ違う補足ポイントを大量生成しない
- 同じ注意喚起文を複数H3に貼り回さない
- 各H3には、その見出し固有の具体例・判断基準・手順・比較・注意点を入れる
- 本文が書けない場合でも、同一文のコピペで埋めない
- 汎用的な安全・コンプライアンス文は記事全体で必要最小限にし、H3ごとに同じ趣旨の段落を繰り返さない

## 水増し見出し・日本語品質の禁止

- 本文文字数を増やすためだけにH2/H3を追加しないでください。
- 本文のないH2/H3、数字だけを変えた見出し、同じH3の繰り返しは禁止です。
- 「セックスアプリを安全に使う実践チェックリスト1〜8」「ケース別の使い分けポイント1〜30」「安全確認の具体例31〜90」「料金と効率の見直しポイント1〜」のような量産見出しは禁止です。
- 元記事より情報量を増やす場合も、検索意図に合う読者に必要な内容だけを追加してください。
- 「サービスサービス」「必要ことです」「注意ことです」「重要ことです」「〜は、〜は、」などの不自然な日本語を出力しないでください。
- 同じ定型文や、同一またはほぼ同一の段落を連続させないでください。
