# バイク買取SEOリライトルール

## 目的

バイク買取MAX / `https://poi-poi.co.jp/bike/` 配下の記事を、検索意図に合うバイク買取SEO記事としてリライトします。元記事の重要なH2/H3、主要情報、検索意図、重要な固有名詞を保持し、読者が自然に査定依頼・問い合わせへ進める構成にしてください。

## 基本方針

- 記事URLから取得した `original.html` を必ず分析する。
- 既存 `original.html` のフォールバックで済ませない。
- 元記事の重要なH2/H3を勝手に削除しない。
- 元記事の主要情報、検索意図、重要な固有名詞を保持する。
- 不要な重複を整理した結果、元記事より短くなることは許容する。
- 文字数比較は警告用途に限定し、文字数差だけでリライト失敗にしない。
- 文字数不足を理由に追加執筆しない。主要情報・検索意図・重要な固有名詞の欠落がある場合だけFAILにする。
- 検索意図を変えない。
- 水増し目的の量産見出し、同一段落の貼り回し、本文のない見出しは禁止する。
- 文字数を合わせる目的の文章追加は禁止する。
- 店舗名、地域名、車種名だけを差し替えた定型文を生成しない。
- 固有情報が不足する項目には一般論を個別に繰り返さない。
- 複数項目に共通する説明は、一覧や比較表の前後に一度だけまとめる。
- 各項目に固有情報を追加できない場合は、既存説明だけで完了するか、比較表へ統合する。
- まとめを最終H2とし、まとめ後に本文や見出しを追加しない。
- WordPressに貼り付け可能なHTMLで作成する。
- SWELLテーマで崩れにくいHTMLにする。
- FAQ構造化データは不要。
- rel=sponsored はショートコード側で管理するため本文では不要。

## キーワード方針

- メインKWは記事URLごとに指定する。
- 指定がない場合は、URL・タイトル・元記事のH1/H2/H3から自然に推定する。
- 共通関連語として「バイク 買取」「バイク 査定」「バイク 売る」「バイク買取 おすすめ」「バイク買取 相場」「出張買取」「一括査定」を必要に応じて補完する。
- 地域記事では「地域名 バイク買取」「地域名 バイク査定」「地域名 原付 買取」を補完する。
- 車種記事では「車種名 買取」「車種名 買取相場」「メーカー名 車種名 売る」を補完する。
- 業者比較記事では「業者名 口コミ」「業者名 評判」「業者名 買取」「業者名 査定」を補完する。

## 必ず意識する内容

- バイクを高く売るコツ
- 査定前にやるべき準備
- 必要書類
- 名義変更・廃車手続きの注意点
- 自賠責保険・軽自動車税・ローン残債の注意点
- 出張査定の流れ
- 店舗持ち込みと出張買取の違い
- 一括査定のメリット・デメリット
- 不動車・事故車・放置バイク・旧車・カスタム車の買取可否
- 査定額が下がりやすいポイント
- 買取業者選びの基準
- トラブルを避けるための注意点
- 契約前に確認すべきこと
- 個人情報の取り扱い
- クーリングオフや契約トラブルの一般的な注意点


## 導入文と入力外部リンクの扱い

- 記事タイトルまたはH1の直後、最初のH2見出しより前に、2〜4文程度の自然な導入文を置く。
- 元記事などの入力データに外部リンクが1件以上ある場合は、記事内容と最も関連性が高い外部リンクを1件選び、導入文の1段落目または2段落目に文章内リンクとして配置する。
- 外部リンクが複数ある場合は、メーカー公式サイト、製品公式ページ、公的機関、一次情報を優先し、記事タイトル・H1・本文テーマに最も近いリンクを選ぶ。
- 導入文には、記事で扱うバイク・製品・サービス・店舗などの概要、読者が得られる情報、リンク先を参照する理由を自然に含める。
- アンカーテキストはリンク先の内容が分かる具体的な文言にし、「こちら」「詳細はこちら」やURL単体は使わない。
- 同じ外部URLが本文中にある場合は、原則として導入文側の1回に集約し、不必要に重複させない。
- 入力データに外部リンクがない場合は、URLを推測したり新規生成したりしない。

## CV導線

- 読者の不安解消を優先し、自然に査定依頼・問い合わせへ進める導線を入れる。
- 唐突な買取訴求や過剰な煽りは禁止する。
- おすすめ業者、比較表、チェックリスト、FAQの後など、読者が判断しやすい位置に導線を配置する。

## 禁止事項

- バイク買取と無関係なジャンルの話題を残さない。
- 文字数を合わせる目的で本文を水増ししない。
- 重要H2/H3を削除しない。
- 空のH2/H3を作らない。
- 数字だけ違う量産見出しを作らない。
- 同じH3を繰り返さない。
- 同じ注意喚起文を複数H3に貼り回さない。
- 「サービスサービス」「必要ことです」「注意ことです」「重要ことです」など不自然な日本語を残さない。
- 認証情報を出力しない。
- `.env` と `node_modules/` をコミットしない。
- 既存公開記事を更新しない。
